Selecting Ad Schedule

1. Introduction

OVA is introducing a number of options that allow various "rules" to be applied during ad scheduling. These rules impact the way ad slots are applied across a playlist. The "applyToParts" rule is the first option that is available.

The "applyToParts" rule can be used to identify the specific clips in a playlist (by index) that will have a particular ad slot "applied to" it. For example, if there is a playlist with four items in it and pre-rolls are only to be applied to the first and last items in the playlist, "applyToParts" would used with the pre-roll ad slot declaration as follows:

{
    ... 
    "ads": {
        "schedule": [
              {
                  "position": "pre-roll",
                  "applyToParts": [0, 3]
              }
        ]
    }
}

"applyToParts" takes a list of playlist index numbers as the value. Playlist items are numbered 0 through to the length of the playlist minus 1, so a four item playlist would have clips as index 0, 1, 2 and 3. As a result, in our example above, the "pre-roll" ad slot is declared as being "applied to" playlist clips at index 0 and 3 (the first and last clips).

"applyToParts" can be used across all ad slot types. For example, if pre-rolls were to be played on the first and last clips in a four clip playlist, and overlays on the second clip in a playlist and a mid-roll on the third clip in the playlist, the following config would be used:

{
    ... 
    "ads": {
        "schedule": [
              {
                  "position": "pre-roll",
                  "applyToParts": [0, 3]
              },
              {
                  "position": "auto:bottom",
                  "startTime": "00:00:10",
                  "applyToParts": [1]
              },
              {
                  "position": "mid-roll",
                  "startTime": "00:00:30",
                  "applyToParts": [2]
              }
        ]
    }
}

It is also possible to have multiple ad slots applied to the same clip. For example, the following config results in pre and post-rolls being applied to the second clip in a playlist:

{
    ... 
    "ads": {
        "schedule": [
              {
                  "position": "pre-roll",
                  "applyToParts": [1]
              },
              {
                  "position": "post-roll",
                  "applyToParts": [1]
              }
        ]
    }
}
