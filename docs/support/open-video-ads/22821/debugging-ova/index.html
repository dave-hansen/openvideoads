Debugging OVA

Purpose

This guide explores how to debug your OVA setup.

It is important to use the debug output generated by OVA at runtime to trace how OVA is interpreting and operating your setup. It is particularly useful to assess how OVA is processing your ad tags. In addition, please reference this output in any postings that you may submit to the OVA forum or the support team as it will help pinpoint any issues that you may be experiencing.

1. Overview

The OVA plugins and frameworks generate a wide range of debug data as they operate. All debug output is exported by OVA via a JavaScript"console.log()" call which will result in the information appearing in the Browser's JavaScript console.

In Chrome, you can view the output via the Chrome "Tools | JavaScript Console or Developer Tools" option. A similar console can be found in IE8/9 via the "F12 Developer Tools". To view the OVA debug output with Firefox, the "Firebug" add-on for Firefox is recommended. Firebug can be found at www.getfirebug.com.

The amount of debug information output by OVA can be filtered via the "debug" config options. One of the most important elements of debug output generated by OVA is the "version" number for your install. 

09:26:28 GMT+0100 Debuggable: OVA for JW 5.x - v1.0.0 RC6 (build 4) - player is 5.7.1896

OVA will output the version number if the "config" debug level is declared in your OVA setup.

2. An Example Setup

To enable OVA debugging you will need to set the "debug" config option in your OVA setup.

<div id="container"></div>

<script type="text/javascript">
    jwplayer("container").setup({
        'flashplayer': 'jwplayer.swf',
        'plugins': {
            'ova-jw': {
                "debug": {
                     "levels": "fatal, config, vast_template, vpaid, http_calls"
                },
                "ads": {
                     "schedule": [
                         {
                             "position": "pre-roll",
                             "tag": "http://adserver/my-ad-tag"
                         }
                    ]
                } 
            }
        }
    });
</script>

3. The Debug Configuration Options

To fine tune and identify the debug information to output, a range of options can be specified in the "levels" configuration option.

The following table identifies the various debug information categories that can be output:

Option Description
all Prints all debug messages
api Prints debug related to the operation of the OVA JS API
clickthrough_events Prints debug messages associated with click through events
cuepoints_events Prints all debug messages fired when cuepoints are triggered
config Prints all debug messages generated as the config data is read
cuepoint_formation Prints all debug messages associated with establishing cuepoints
display_events Prints all debug messages associated with non-linear ad display events
fatal Prints all debug messages marked as fatal
http_calls Prints all debug messages generated as HTTP calls are made
mouse_events Prints all debug messages associated with mouse events
none Turns off all debug output
playlist Prints all debug messages generated as playlists are manipulated
region_formation Prints all debug messages associated with the construction of regions
segment_formation Prints all debug messages generated when the stream sequence is formed
tracking_events Prints all debug messages tied to tracking events
tracking_table Prints all debug messages tied to the tracking table
vast_template Prints debug messages generated during requesting and parsing the VAST ad data
vpaid Prints debug related to the operation of VPAID ads
