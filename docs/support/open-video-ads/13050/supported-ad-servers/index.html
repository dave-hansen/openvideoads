1. Introduction

In short, Open Video Ads will work with any VAST (1 or 2) compliant video ad server or provider. The following list covers just some of the VAST ad providers that are known to work with OVA.

2. Connecting Directly with Static Ad Tags

As a rule of thumb, a VAST ad provider generally provides what is known as a static ad tag to use with OVA. A static ad tag is a predefined URL that when called, delivers a VAST response in XML form.

Static ad tags are the easiest type of ad tag to use with OVA. In nearly all cases, you will configure OVA with a static ad tag. The following are examples of static ad tags for a variety of ad server types:

http://track.adform.net/serving/videoad/?bn=45xxxx&ord=1319031161332

http://ad.afy11.net/ad?enc=4&asId=10000043xxxx7&sf=0&ct=256

http://ad.doubleclick.net/pfadx/N270.126913.6102203221521/B3876671.21;dcadv=221xxx;sz=0x0;ord=__timestamp__;dcmt=text/xml

http://ad3.liverail.com/?LR_PUBLISHER_ID=1331&LR_CAMPAIGN_ID=228&LR_SCHEMA=vast2

Although static ad tags are predefined by the ad provider, a number of Ad Tag Variables (e.g. __timestamp__) may be used with the ad tag to allow OVA to place run-time specific information directly into the URL as it is called - for example, the URL of the page running the player, the referrer, a random-number to break the cache or a timestamp. Please consult the Using OVA Tag Variables guide for further information on this topic.

Static ad tags are best used with OVA via the tag config option. The following code snippet illustrates a typical setup for a pre-roll ad position with a static ad tag that includes the referrer OVA ad tag variable:

{
    "ads": {
          "schedule": [
                {
                     "position": "pre-roll"
                     "tag": "http://ad.afy11.net/ad?enc=4&asId=10407&referrer=__referrer__"
                }
          }
    }
}

For more information regarding static ad tags, please consult the Using Static Ad Tags tutorial.

3. Support for Dynamic Ad Tags

A few providers however encourage the ad tag URL to be "dynamically" generated by OVA at runtime based on a URL template for the specified ad server "type". This is useful if the ad server supports fulfilling multiple ad positions in a single request. OVA directly supports a number of these ad servers "out-of-the-box" - namely OpenX, AdTech, 24/7 Real Media and OASIS. The most obvious example of an ad server that uses the dynamic ad tag feature is the "OpenX" Community Edition. The following code snippet illustrates an example OpenX OVA config:

{
    "debug": {
          "levels": "fatal, config, vast_template"
    },
    "ads": {
           "servers": [
               {
                   "type": "OpenX",
                   "apiAddress": "http://openx.openvideoads.org/openx/www/delivery/fc.php"
               }
          ],
          "schedule": [
               {
                    "zone": "5",
                    "position": "pre-roll"
               },
               {
                    "zone": "33",
                    "position": "auto:bottom",
                    "startTime": "00:00:10"
               }
          ]
    }
}

In this example, a single ad request is formatted for a pre-roll (zone 5) and overlay ad (zone 33). The ad tag constructed by OVA would look like:

http://openx.openvideoads.org/openx/www/delivery/fc.php?script=bannerTypeHtml:vastInlineBannerTypeHtml:vastInlineHtml&zones=pre-roll:0.0-0%3D5%7Cbottom:1.0-0%3D33&nz=1&source=&r=R0.9885416775941849&block=1&format=vast&charset=UTF-8

If you find that your ad server is not on the supported list and does not work with OVA via static ad tags, please contact enquiries@openvideoads.org and we will arrange to add support directly for your ad server.
