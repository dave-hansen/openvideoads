1. Introductions

By default, OVA supports click-throughs on linear ads streams.

A click-through is a call to action made to the user as they watch the linear ad. For non-interactive ad streams the call to action comes in the form of a "Click for More Information" notice that appears on the player screen when the user mouses over the player.

For interactive (VPAID) linear ads, the call to action may come in a more obvious manner via some type of interactive display on the player as the ad progresses.

No additional configuration is required to enable click-throughs - OVA automatically configures the click-through process based on the detail provided in the VAST response to an ad call.

2. Working Example

The following instance illustrates the standard click-through notice "Click for More Information" as it appears when the view mouses over the linear ad stream.

{ "ads": { "schedule": [ { "position": "pre-roll", "tag": "OVA_AD_TAG_LINEAR_STREAM_WITH_80x300_COMPANION" } ] }, "debug": { "levels": "fatal, config, vast_template" } }
{ "playlist": [ { "url": "OVA_HTTP_SHOW_STREAM" } ], "plugins": { "controls": { "autoHide": "never" }, "ova": { "url": "OVA_FLOWPLAYER_PLUGIN_SWF" } } }
{ "flashplayer": "OVA_JW5_PLAYER_SWF", "playlist": [ { "file": "OVA_HTTP_SHOW_STREAM" } ], "width": 450, "height": 300, "controlbar": { "position": "bottom" }, "plugins": { } }

3. Modifying the Click-Through Message

The appearance of the click-through message can be modified via the clickSign configuration block as follows:

{
     ...
     "ads": {
          "clickSign": {
                 "enabled": true, 
                 "verticalAlign": "center",
                 "horizontalAlign": "center",
                 "width": 150,
                 "height": 32,
                 "opacity": 0.5,
                 "borderRadius": 20,
                 "backgroundColor": "#000000",
                 "style": ".smalltext { font-size:12; }",
                 "html": "<p class="smalltext" align="center">CLICK ME!</p>",
                 "scaleRate": 0.75
          },
          .....
}

The click-through message is presented as a display region on the player screen. As such, it can be fully customised by declaring the standard properties of a region within a clickSign block. For more information on customising a region, please refer to the Customising Display Regions guide.

In the example config above, the wording of the click-through message is changed to "CLICK ME!" and that wording is placed in the center of the player screen with a slightly tinted black background.

4. Pausing on Click-Through

By default playback is paused when the user clicks through on linear ad.

OVA be instructed to keep playing after the viewer clicks-through via the pauseOnClickThrough configuration variable.

The following code snippet illustrates how to use pauseOnClickThrough:

{
     ...
     "ads": {
          "pauseOnClickThrough": false
          .....
}

5. Modifying the Target Window

When the viewer clicks through, the click-through URL that has been declared in the VAST response will be loaded in a new tab in the browser. It is possible to modify the target window used to load that click-through URL via the clickSign.target configuration variable. The following code snippet illustrates how to change the target window to the one containing the player:

{
     ...
     "ads": {
          "clickSign": {
               "target": "_self"
          }
          .....
}

This option can be seen in action here.

Valid options are "_self", "_blank", "_parent" and "_top". The default setting is "_blank".

6. Turning off the Click-Through

The click-through mechanism can be turned off via the "clickSign.enabled" option as follows:

{
     ...
     "ads": {
          "clickSign": {
               "enabled": false
          }
          .....
}

7. Tracking Clicks

By default, all click-through actions are tracked by OVA in accordance with the specification provided in the VAST response.

It is possible to also track the actions via Google Analytics using the OVA GA customisations - please refer to the Using Google Analytics with OVA guide for further information.
