1. Introduction

OVA is capable of supporting the notion of an ad "story" that is a sequence of linear ads that have to be played in a particular order to tell a "story" to the viewer. The most direct form that an ad story can take is a series of "pre-rolls" that are played sequentially before a main stream. More complex forms that the form of pre, mid and post-rolls that are spread across the stream but have to be played in a particular order.

To configure an ad story, three OVA config option groups have to be used together in a specific way. The two option groups are:

  1. The AdSlot "repeat" option (and potentially the optional "interval" option), along with
  2. The ad tag config options within a "servers" or the ad slot "server" grouping
  3. The "oneAdPerRequest" option

2. A Single Position Ad Story

To deliver a series of related ads in one specific pre, mid or post-roll position, the "repeat" option should be used as follows:

{
    ... 
    "ads": {
        "schedule": [
              {
                  "position": "pre-roll",
                  "repeat": 3,
                  "server": {
                      "tag": "your-ad-tag-goes-in-here",
                      "oneAdPerRequest": false
                  }
              }
        ]
    }
}

In this example, because "oneAdPerRequest": false is specified, the three repeated pre-roll ad slots will share a single ad call. For that ad call to deliver an ad story, it must deliver enough inventory to fill the three ad slots. Any ad slots that are not filled will not be played.

It is also possible to declare the same setup as follows:

{
    ... 
    "ads": {
        "schedule": [
              {
                  "position": "pre-roll",
                  "repeat": 3,
                  "tag": "your-ad-tag-goes-in-here",
                  "oneAdPerRequest": false
              }
        ]
    }
}

4. Running a Story Over Multiple Mid-Rolls

OVA for Flowplayer has a limitation at present that means that when delivering multiple sequential mid-rolls via the "repeat" option, the mid-roll positions need to be spaced at least 2 seconds apart using the "interval" option to specify the spacing in terms of a specific number of seconds. The following config illustrates how to deliver a mid-roll ad story that has 3 mid-roll positions being delivered 30 seconds apart:

{
    ... 
    "ads": {
        "schedule": [
              {
                  "position": "mid-roll",
                  "repeat": 3,
                  "interval": 30,
                  "tag": "your-ad-tag-goes-in-here",
                  "oneAdPerRequest": false
              }
        ]
    }
}

4. Supporting Multiple Ad Stories

It is also possible to deliver multiple ad stories across an ad schedule. For example, the following config snippet illustrates how to deliver an ad story in the pre-roll position across 3 ad slots, a different ad story in the mid-roll position across 2 mid-roll ad slots, and a third story in the post-roll position across 3 ad slots.

{
    ... 
    "ads": {
        "schedule": [
              {
                  "position": "pre-roll",
                  "repeat": 3,
                  "tag": "your-first-ad-tag-goes-in-here",
                  "oneAdPerRequest": false
              },
              {
                  "position": "mid-roll",
                  "startTime": "00:00:30",
                  "repeat": 2,
                  "interval": 30,
                  "tag": "your-second-ad-tag-goes-in-here",
                  "oneAdPerRequest": false
              },
              {
                  "position": "post-roll",
                  "repeat": 3,
                  "tag": "your-third-ad-tag-goes-in-here",
                  "oneAdPerRequest": false
              },
        ]
    }
}
