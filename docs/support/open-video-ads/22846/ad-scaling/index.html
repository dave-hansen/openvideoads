1. Introductions

OVA scales linear and non-linear ads automatically based on the scaling attributes provided in the VAST response. It is possible to override the automatic scaling rules and forcibly set the scaling attributes for all linear or non-linear ads.

2. Scaling Linear Ads

2.1 Automatic scaling based on the VAST attributes

OVA will automatically scale linear ads to "fill" the player where possible.

A VAST response however may specify that a linear ad is not to be scaled via the "scalable" attribute on the media file tagset. For example:

<MediaFile scalable="false" width="640" height="480" type="video/x-mp4">
    <URL>
       <![CDATA[http://static.bouncingminds.com/ads/30secs/country_life_butter_600.mp4]]>
    </URL>
</MediaFile>

If a linear ad has scalable="false" set in the VAST response, the ad will not be scaled by OVA

2.2 Forcing scaling

A configuration option has been added to allow the linear scaling attributes of ad streams to be forcibly set.

To forcibly set the linear scaling attributes, use the linearScaling as follows:

{
   ...
      "ads": {
          "linearScaling": "value-goes-in-here",
          "schedule": [
               ...
          ]
      }
   ...
}

linearScaling takes a string value. The value is dependant upon the player implementation.

For Flowplayer, the value can be either:

  • "orig"
  • "fit"
  • "scale"

For JW Player, the value can be either:

  • "uniform"
  • "fill"
  • "exactfit"
  • "none"

3. Scaling Non-Linear Ads

By default, OVA will not scale non-linear ads unless they are VPAID ads. To scale image and non-VPAID SWF non-linear ads types, scaling must be explicitly turned on.

Regardless of whether or not OVA scaling is enabled, scaling will not be activated unless the non-linear VAST tags specify that the asset is "scalable". For example, the following VAST response specifies that the non-linear image is scaled:

<NonLinear id="12" width="728" height="90" scalable="true"> 
   <StaticResource creativeType="image/gif"> 
      http://static.openvideoads.org/ads/overlays/yu-gi-oh-728x90.gif
   </StaticResource>
</NonLinear>

3.1 Enabling Scaling on Image and Non VPAID SWF Non-Linear Ads

To enable scaling on image and non-VPAID SWF non-linear ad types, use the "enableScaling" option as follows:

{
    "ads": {
        "overlays": {
           "enableScaling": true
        },
        "schedule": [
           {
              "region": "auto:bottom",
              "startTime": "00:00:03",
              "duration": "recommended:15",
              "tag": "http://ad.server.com/my-ad-tag"
           }
        ]
    }
}

It is also possible to declare scaling for specific ad slots as follows:

{
    "ads": {
        "schedule": [
           {
              "region": "auto:bottom",
              "startTime": "00:00:03",
              "duration": "recommended:15",
              "enableScaling": true,
              "tag": "http://ad.server.com/my-ad-tag"
           }
        ]
    }
}

3.2 Scaling to Specific Dimensions

When OVA scales images and SWFs, by default it will try to scale the assets to the dimensions of the player. It is possible to instruct OVA to scale to specific dimensions as follows:

{
    "ads": {
        "schedule": [
           {
              "region": "auto:bottom",
              "startTime": "00:00:03",
              "duration": "recommended:15",
              "enableScaling": true,
              "width": 300,
              "height": 50,
              "tag": "http://ad.server.com/my-ad-tag"
           }
        ]
    }
}

3.3 Maintaining Aspect Ratio

When OVA scales image and SWF assets, it will attempt to maintain the aspect ratio of the assets if the "maintainAspectRatio" attribute is set on the non-linear VAST ad tags as follows:

<NonLinear id="12" width="728" height="90" scalable="true" maintainAspectRatio="true"> 
   <StaticResource creativeType="image/gif"> 
      http://static.openvideoads.org/ads/overlays/yu-gi-oh-728x90.gif
   </StaticResource>
</NonLinear>

4. Disabling OVA Initiated Scaling of VPAID ads

If a VAST response has "scalable=true" set, then OVA will automatically attempt to scale the VPAID SWF resource to fit the player dimensions.

At times, this forced scaling impacts the way VPAID ads internally scale often resulting in a VPAID ad not rendering correctly.

It is possible to disable OVA auto-scaling via the following options:

...
      "ads": {
           "vpaid": {
               "enableLinearScaling": false,
               "enableNonLinearScaling": false
           },
           .....
